Migrating Everything To Redux Toolkit
-------------------------------------
For dispatching actions, createSlice ... covered.
It automatically creates unique action identifiers for our different reducers. <------------- **

To get hold of these action identifiers we can use our counterSlice ...

counterSlice.actions.toggleCounter() <------------ Will create action object 
returns an action object of this shape:
{ type: "Some auto-generated unique identifier" }

... these methods are called action creators 
... these objects already have a type property with a unique identifier per action

store/index.js
--------------
...
export const counterActions = counterSlice.actions;
export default store;

Counter.jsx
-----------
...
  dispatch(counterActions.increment());

What do we do here, when we also need a playload?
...
  dispatch({ type: "INCREASE", payload: 10 }); <----------- *
  dispatch(counterActions.increase(10));

... how you then extract that value, ... Redux Toolkit automatically create action objects 
... any value you pass here as an arugment ... will be stored in an extra field name payload.

{ type: SOME_UNIQUE_IDENTIFIER, payload: 10 }
